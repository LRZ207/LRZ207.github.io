Wonderful War World 部署计划书

WonderfulWarWorld部署计划：

第一部分（总体规划）

WonderfulWarWorld模拟的是现代战争，主要以海战为主（包括岛屿的抢滩登陆和港口战役），在第二阶段将会全面支持现代战争，在专业版将会支持“协议”内的mod和资源，目前规划的WonderfulWarWorld将会支持网络模式（但需要自主架设服务器），预计2.5阶段可以支持快速网络模式即p2p内网穿透传输。 目前第一阶段的最低硬件要求：

E系列奔腾处理器，512MB系统内存，板载集成显卡，WindowsXP，1GB磁盘空间

可选项：

更好的显卡，CPU，更多内存以及磁盘空间，网络速度达到5Mb以上

第二阶段：

corei3-320，512MB游戏内存，板载显卡，Windows7，2GB磁盘空间，要求系统必须支持.net3.0以上。

可选项： 同第一阶段可选项（网络到达10Mb以上） 第三阶段： core i3，2GB系统内存，HD4000核芯显卡，Windows7及以上（建议Windows10或11），3GB磁盘空间，系统需要支持.net4.0以上版本
可选项：同第二阶段可选项（网络达到15Mb以上）

第二部分（开发部分）
第一阶段：使用Visual Basic和C++编写，图像图形全部使用贴图和简易功能实现,内部接口全部使用“秦”级，网络部分使用WINSOCK部件实现的tcp和udp协议。为了能够尽可能缩短时间，部分细节会暂时粗糙化，通过补丁更新实现完善。
开发软件：Visual Basic 6.0 Visual C++ 6.0
Visual Studio 2019 Visual Studio Code(编程部分)
Photoshop 2022 (图形部分)
第二阶段：使用Visual C++编写，重要图像全部用DX引擎渲染，余下部分使用细腻贴图，内部接口改用“汉”级接口，部分支持多线程（“汉”级仍然是“秦”级的改进版，注意：这里只保留了“秦”级的设计理念和逻辑，在算法上和开发兼容性已有很大区别），网络通讯方式采用WIN点对点技术进行实验SOCKapi实现并进行部分重封装，将p2p技术实现。
第三阶段：使用Visula C++编写，利用第二阶段的DX渲染框架作为基础，放弃绝大部分的贴图文件。
签名：207（批准核验APPROVE）
第三部分
游戏资源算法：传统可执行文件内部实现+含有特定协议的api接口（即“秦”级等接口协议），使用dlls返回资源文件目录和建模目录（含配方式）目前“秦”采用了装载协议，理论上可以支持在协议内的所有资源。
补充：“秦”级以及其他接口将会预留mod的高级扩展功能，在不同的接口下除了“秦”级以外，剩余的接口理论上可以通过兼容设置支持前期mod扩展。
